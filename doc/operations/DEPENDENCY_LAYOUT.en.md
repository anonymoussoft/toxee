# Dependency Layout
> Language: [中文](DEPENDENCY_LAYOUT.md) | [English](DEPENDENCY_LAYOUT.en.md)

This document describes the third-party dependency layout used by toxee and how a fresh clone is made buildable via the bootstrap workflow.

## Current assumptions that break a fresh clone (legacy)

Before bootstrap was introduced, a fresh clone of toxee could not resolve dependencies because:

- **tim2tox_dart** was only resolvable through `../tim2tox/dart` (sibling directory outside the repo).
- **UIKit packages** (tencent_cloud_chat_common, tencent_cloud_chat_message, etc.) were only resolvable through `../chat-uikit-flutter/...`.
- **tencent_cloud_chat_sdk** was only resolvable through `../tencent_cloud_chat_sdk-8.7.7201`.

All of these paths assume manually prepared sibling repositories. The bootstrap workflow removes this requirement.

## Target layout (after bootstrap)

```
toxee/
├── third_party/
│   ├── tim2tox/                 # git submodule (https://github.com/anonymoussoft/tim2tox)
│   │   ├── tool/
│   │   │   ├── tencent_cloud_chat_sdk.lock.json   # SDK version + archive URL
│   │   │   └── apply_sdk_patches.dart             # apply patch series to SDK dir
│   │   └── patches/tencent_cloud_chat_sdk/<ver>/  # series + *.patch files
│   ├── chat-uikit-flutter/      # git submodule (anonymoussoft/chat-uikit-flutter fork)
│   └── tencent_cloud_chat_sdk/  # generated: downloaded + patched vendor tree
├── tool/
│   ├── bootstrap_deps.dart      # bootstrap CLI (orchestrates submodules, vendor, calls tim2tox apply)
│   └── vendor_state.json        # generated
└── pubspec_overrides.yaml       # generated; points all local deps into third_party/
```

- **third_party/tim2tox** (upstream [https://github.com/anonymoussoft/tim2tox](https://github.com/anonymoussoft/tim2tox)) and **third_party/chat-uikit-flutter** are git submodules. They are initialized and updated by `dart run tool/bootstrap_deps.dart`.
- **third_party/tencent_cloud_chat_sdk** is not in version control. It is downloaded from a locked archive and patched by the bootstrap tool.
- **pubspec_overrides.yaml** is generated by the bootstrap tool so that `flutter pub get` resolves tim2tox_dart, tencent_cloud_chat_sdk, and all UIKit packages from paths under `third_party/` only.

## Entry flow for a fresh clone

1. `git clone <toxee-repo>`
2. `dart run tool/bootstrap_deps.dart` — initializes submodules, vendors and patches the SDK, writes `pubspec_overrides.yaml`
3. `flutter pub get`
4. Build (e.g. `./build_all.sh --platform macos --mode debug`)

See [DEPENDENCY_BOOTSTRAP.en.md](DEPENDENCY_BOOTSTRAP.en.md) for the exact bootstrap order and [PATCH_MAINTENANCE.en.md](PATCH_MAINTENANCE.en.md) for ownership and upgrade workflows.
